Core Deliverables

The department owns five primary service areas, each with defined responsibilities and escalation protocols.

🔍

Technical SEO Audits

2.1 — Monthly site-wide health check  ·  View full fix list →

  • Crawl all client websites monthly (Screaming Frog or equivalent)
  • Identify & resolve indexation issues — noindex tags, canonical errors, blocked pages
  • Detect and fix broken internal and external links (4xx, 5xx)
  • Identify and resolve duplicate content issues
  • Audit and clean redirect chains (3xx redirects)
  • Review XML sitemaps and robots.txt for errors
  • Flag thin content pages for refresh or consolidation
  • Escalate server-side and hosting issues to the developer team
📄

On-Page SEO Updates

2.2 — Per page SOP, quantity depends on package

Each page goes through the following steps. Number of pages per month is determined by the client's package (e.g. Starter = 2 pages/month). This task does not include content expansion or rewriting beyond headings and metadata.

  • Review each page and identify main keywords
  • Update Title Tag & Meta Description
  • Optimize Headings (H1–H3)
  • Review and improve URL structure (only if necessary)
  • Add/update internal links from relevant pages
  • Improve Page Speed and Core Web Vitals
  • Perform a final check for mobile usability and keyword placement
  • Complete reporting and mark the task as complete
✍️

Content Strategy & Gap Analysis

2.3 — Capture demand, fill gaps

  • Monthly keyword gap analysis using Ahrefs or SEMrush
  • Identify high-value topics the site is not currently targeting
  • Recommend new service pages or location pages based on demand
  • Monthly blog topic recommendations by search intent and competition
  • Structured content briefs for writers: keyword, intent, outline, word count, links
  • Review published content to ensure briefs were followed
  • Monitor performance in GSC and adjust strategy accordingly
  • Flag underperforming pages for refresh or consolidation
🏷️

Schema Markup

2.4 — Structured data & rich results

  • Audit existing schema markup across all client pages
  • Implement LocalBusiness schema for local SEO clients
  • Add schema types: FAQPage, HowTo, Article, Review, MedicalCondition
  • Validate all schema using Google's Rich Results Test
  • Monitor GSC for schema-related errors and warnings
  • Update schema when business information changes
  • Stay current with Google's schema documentation
⚡

Core Web Vitals & Page Speed

2.5 — Google ranking factor; audit, fix, and escalate

  • Monthly CWV reports via PageSpeed Insights and Google Search Console
  • Identify underperforming pages (LCP, INP, CLS below "Good" threshold)
  • Implement quick wins: image compression, lazy loading, render-blocking flags
  • Create detailed escalation tickets for developers: URL, failing metric, fix
  • Follow up with developers and re-test after implementation
  • Track CWV improvements over time and include in monthly reports
  • Flag hosting or server response time issues for client or developer review

Reporting & Optimization Roadmap

Monthly Report Contents

  • Keyword ranking summary — top 10 tracked keywords with position changes
  • Organic traffic overview: sessions, impressions, clicks from GSC
  • Technical SEO health: issues found, resolved, still open
  • Core Web Vitals status: pass/fail per page with trend
  • Content performance: top pages by traffic, new content metrics
  • Completed work this month: all deliverables executed
  • Top 10 optimization priorities for next month

Optimization Roadmap

The roadmap is a prioritized living list of the ten highest-impact actions for the coming month. It is shared with the account manager and client before work begins, ensuring full transparency and client buy-in.


Roadmaps are built from three inputs: technical audit findings, keyword performance data from GSC, and content gap analysis. The goal is always the highest ROI actions first — typically fixing crawl issues, then targeting quick-win keywords, then expanding content.

Team Structure at Scale

As the agency grows from 100 to 400+ clients, the department scales in a predictable pattern. Specialists keep 50–70 clients each; leadership and specialization layers are added as volume increases.

Client Volume Recommended Team Structure
Up to 100 clients 2–3 generalist SEO specialists + shared link building team
100–200 clients 3–4 specialists + 1 dedicated local SEO / GBP person
200–300 clients 5–6 specialists + 1 local SEO lead + content team
300–400+ clients 7–8 specialists + team leads + content writers + developer liaison
At 300+ clients, introduce a Department Lead responsible for QC, training, process documentation, and escalation management.
